# ICF 修订记录工具使用说明

## 快速启动

1. 解压整个压缩包到本地目录。
2. 双击 `RUN_WEB.bat`。
3. 浏览器打开：

```text
http://127.0.0.1:8000
```

首次启动会自动创建 `.venv` 并安装依赖。机器需要提前安装 Python 3.12。

## 日常使用

1. 上传带 Word 修订痕迹的 ICF `.docx`。
2. 模板路径保持默认：`TG-ICF模板\标准模板.docx`。
3. 没有验收文件时选择 `draft`。
4. 输出路径建议写入 `output\`，例如：

```text
output\case-001-draft.docx
```

5. 下载生成结果并人工复核。

## 交付状态解释

- `draft_generated`：草稿已生成，只能用于人工复核。
- `release_passed`：模板完整性和验收文件严格校验均通过，可作为正式交付。
- `release_blocked`：文件已生成，但与验收文件仍有差异，不能直接交付。

## 当前限制

1. 当前只支持包内 `TG-ICF模板\标准模板.docx`。
2. 没有验收文件时不能宣称正式交付。
3. 自动归类仍需人工复核，尤其是主题、页码、原因和表格内容。
