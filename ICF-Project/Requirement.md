# Requirement

## 1. 项目背景

本项目服务于 CRA 和相关医学文档协作人员。  
目标是减少人工把 ICF 修订内容录入《文件修订记录》模板的时间，并保证正式交付结果与验收文件达到可直接交付级一致。

## 2. 核心目标

1. 读取带 Word 修订痕迹的 ICF `.docx`。
2. 提取修订前文本、修订后文本、位置、来源和样式信息。
3. 依据稳定规则把修订内容整理成交付模型所需的业务行。
4. 在标准模板上通过 OOXML 原生定点写入生成 `.docx` 结果。
5. 以验收文件作为离线金标准执行严格交付校验，但不得用验收文件行回放冒充源文档自动生成结果。
6. 将稳定规则沉淀为可复用的 skill 和项目文档。

## 3. 目标用户

1. CRA
2. 医学写作或项目管理人员
3. 本地运行和复核文档的内部团队

## 4. 输入

1. 标准模板：`TG-ICF模板/标准模板.docx`
2. 训练集：
   - `训练集/训练文件`
   - `训练集/训练文件验收文件`
3. 测试集：
   - `测试集/测试文件`
   - `测试集/验收文件`
4. 用户通过 Web 或 skill 提供的源文件、模板路径、输出路径和运行模式
5. `release` 模式下必须提供验收文件，或由系统自动定位到配对验收文件

## 5. 输出

1. 模板化的修订记录文档：`*.docx`
2. 严格交付报告：`*.report.json`
3. 交付差异报告：`*.acceptance_diff.json`
4. 复盘文档：`output/progress/*.md`
5. 可复用 skill 安装包：`dist/icf-amendment-history.skill`

## 6. 功能需求

### 6.1 文档读取

1. 程序必须读取正文、表格、页眉、页脚和运行级样式信息。
2. 程序必须识别插入、删除、替换等 Word 修订痕迹。
3. 程序不能丢失纯新增或纯删除修订。

### 6.2 修订抽取与交付建模

1. 删除内容写入“原文”列，并保留删除线语义。
2. 新增内容写入“修订后内容”列，并保留新增强调语义。
3. 替换型修订需要同时保留旧文本和新文本。
4. 交付模型必须保留：
   - 主题
   - 修订章节/页码
   - 原文
   - 修订后内容
   - 更改原因
   - 关键样式标记
   - 文本或表格富内容块

### 6.3 业务归类

1. “主题”需要依据修订实际落点进行业务归类。
2. “修订章节/页码”必须体现章节名和页码范围。
3. “更改原因”需要贴近验收文档的稳定措辞风格。
4. 页脚版本号/日期修订默认位于首行。
5. 首页中心信息默认合并为摘要行。
6. 表格修订必须保留表格来源痕迹。

### 6.4 模板写回

1. 输出必须建立在标准模板上，不允许自由拼装文档骨架。
2. 非白名单模板结构不得被破坏。
3. 只允许修改题头版本值和修订记录表数据行。
4. 非 `word/document.xml` 的 package part 必须字节级保持不变。
5. `[表格]` 类内容必须以表格 OOXML 嵌入单元格，不得仅转成纯文本。

### 6.5 产品交付

1. 提供本地 Web 界面，支持文件上传、模板路径、验收文件路径、输出路径、模式切换、日志和结果下载。
2. 提供 skill 脚本入口，至少支持：
   - `generate`
   - `validate`
   - `compare`
3. 提供两种运行模式：
   - `draft`
   - `release`

## 7. 非功能需求

1. 本地运行，不依赖广告型或低代码外部平台。
2. 技术方案应可控、可回归、可解释。
3. 模板保护必须可验证。
4. 规则必须优先泛化，不允许为单一样本文案硬编码。
5. 文档和产物要便于后续迭代和审计。

## 8. 验收要求

### 8.1 正式交付

`release` 模式下，正式交付必须同时满足：

1. 模板完整性校验通过
2. 验收文件存在
3. 规范化交付行总数一致
4. 行顺序一致
5. 关键字段无差异
6. 关键样式无差异
7. 无 `missing_rows`
8. 无 `unexpected_rows`

任一条件不满足时，系统必须生成 `release_blocked`，并保留差异报告供复核。

### 8.2 诊断对齐

结构化 alignment 分数仅作为诊断信息保留，用于定位：

1. 主题偏差
2. 章节/页码偏差
3. 更改原因偏差
4. 表格来源痕迹偏差
5. 样式表达偏差

### 8.3 草稿模式

1. `draft` 模式允许无验收文件运行。
2. `draft` 模式不得宣称正式交付。
3. `draft` 模式输出应显式标记 `draft_only=true`。

## 9. 当前约束

1. 当前只支持标准模板，不扩展多模板适配。
2. 当前以 `.docx` 为唯一正式输入输出格式。
3. 验收文件是开发、回归和发布门禁中的离线金标准；没有验收文件时只能输出草稿。
